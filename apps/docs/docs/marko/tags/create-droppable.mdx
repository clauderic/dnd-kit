## Usage

The `<create-droppable>` tag requires a unique `id`. It provides a body parameter with reactive state and a `setElement` function for connecting the drop target element.

```marko
<drag-drop-provider onDragEnd(event) {
  if (!event.canceled) {
    parent = event.operation.target?.id ?? null;
  }
}>
  <create-draggable|draggable| id="draggable">
    <button/$dragEl>
      <lifecycle
        onMount() { draggable.setElement($dragEl()); }
        onDestroy() { draggable.setElement(undefined); }
      />
      draggable
    </button>
  </create-draggable>
  <create-droppable|droppable| id="droppable">
    <div/$el
      class=("droppable" + (droppable.isDropTarget ? " active" : ""))
    >
      <lifecycle
        onMount() { droppable.setElement($el()); }
        onDestroy() { droppable.setElement(undefined); }
      />
      Drop here
    </div>
  </create-droppable>
</drag-drop-provider>
```

## Input

<ParamField path="id" type="UniqueIdentifier" required>
  A unique identifier for this droppable instance.
</ParamField>

<ParamField path="accept" type="string | string[]" optional>
  The types of draggable elements this droppable accepts.
</ParamField>

<ParamField path="type" type="string" optional>
  The type of this droppable element.
</ParamField>

<ParamField path="collisionDetector" type="CollisionDetector" optional>
  A custom collision detection algorithm.
</ParamField>

<ParamField path="disabled" type="boolean" optional>
  Whether the droppable is disabled.
</ParamField>

<ParamField path="data" type="Data" optional>
  Custom data to attach to this droppable instance.
</ParamField>

## Output (body parameter)

<ResponseField name="isDropTarget" type="boolean">
  Whether this element is currently a drop target (a draggable is hovering over
  it).
</ResponseField>

<ResponseField name="setElement" type="(el: Element | undefined) => void">
  Connect a DOM element to this droppable. Call in `onMount` with `$el()`, and
  in `onDestroy` with `undefined`.
</ResponseField>
