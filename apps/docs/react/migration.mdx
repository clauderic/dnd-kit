## Event Monitoring

The `useDndMonitor` hook has been updated to work with the new event system. Here are the key differences:

### Before
```tsx
import {useDndMonitor} from '@dnd-kit/core';

function DragMonitor() {
  useDndMonitor({
    onDragStart({active}) {
      console.log('Started dragging', active.id);
    },
    onDragMove({active, over}) {
      console.log('Dragging over', over?.id);
    },
    onDragCancel({active}) {
      console.log('Drag cancelled', active.id);
    },
    onDragEnd({active, over}) {
      console.log(`Dropped ${active.id} onto ${over?.id}`);
    }
  });

  return null;
}
```

### After
```tsx
import {useDndMonitor} from '@dnd-kit/react';

function DragMonitor() {
  useDndMonitor({
    onDragStart(event) {
      const {source} = event.operation;
      console.log('Started dragging', source.id);
    },
    onDragMove(event) {
      const {source, target} = event.operation;
      console.log('Dragging over', target?.id);
    },
    onDragEnd(event) {
      const {operation, canceled} = event;

      if (canceled) {
        console.log('Drag cancelled', operation.source.id);
        return;
      }

      if (operation.target) {
        console.log(`Dropped ${operation.source.id} onto ${operation.target.id}`);
      }
    }
  });

  return null;
}
```

Key differences:
- Event handlers now receive the `event` object and `manager` instance
- The `active` and `over` properties are now accessed through `event.operation.source` and `event.operation.target`
- `onDragCancel` is replaced by checking `event.canceled` in the `onDragEnd` handler
- New events are available:
  - `onBeforeDragStart`: Fires before drag begins, can be prevented
  - `onCollision`: Fires when collisions are detected with droppable targets
- Event data includes additional information like `nativeEvent` and position coordinates
